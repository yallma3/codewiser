# Specs

This directory defines **what** the agents are building and **how** the system is wired.
- `product.md` — User stories, acceptance criteria, business logic. May reference `product_<brd>.md` files for larger projects.
- `product_<brd>.md` — Business requirements documents scoped to a specific domain or feature (e.g., `product_payments.md`, `product_auth.md`).
- `ux.md` — Design system, personas, persona profiles, and UX requirements for each persona.
- `system.md` — Architecture, component boundaries, schemas, API mappings.
- `system_<module>.md` — Module-scoped system specs for larger projects (e.g., `system_auth.md`, `system_api.md`).
- `spec-index.json` — Machine-readable manifest linking code files to tests and specs.
